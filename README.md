# ACH-Experiment

Имитационная среда для метода адаптивного консистентного хеширования (ACH) с
ограничением стоимости перестроек в гетерогенных распределённых СУБД.

Сопровождает статью:

> Батанов А. О., Болбаков Р. Г. **Метод адаптивного консистентного хеширования
> с ограничением стоимости перестроек для гетерогенных распределённых
> СУБД.**

Авторы: А. О. Батанов
([ORCID 0009-0004-6968-1139](https://orcid.org/0009-0004-6968-1139)),
Р. Г. Болбаков.

---

## Что здесь есть

Репозиторий содержит:

- полную реализацию алгоритма ACH (A4) и четырёх алгоритмов сравнения
  (A0 — Static-W, A1 — Static-Wt, A2 — Dynamic-R, A3 — Bounded-Loads);
- единый имитационный стенд (`src/`) с моделью кольца, кластера, нагрузки,
  телеметрии и шумов;
- конфигурации серий экспериментов C1–C5 (включая подсерии C4a/C4b/C4c) в
  формате YAML;
- скрипты для прогона, агрегации, статистического анализа (доверительные
  интервалы, парный критерий Уилкоксона, дельта Клиффа), генерации графиков;
- юнит-тесты для проверки инвариантов кольца, гистерезиса и
  бюджетного ограничения ACH;
- инструкции для воспроизведения всех результатов статьи.

## Структура

```
ach-experiment/
├── src/                       # Реализация алгоритмов и имитационной среды
│   ├── algorithms/            # ACH и четыре baseline-алгоритма
│   ├── ring.py                # Кольцо консистентного хеширования
│   ├── cluster.py             # Модель гетерогенного кластера
│   ├── load_generator.py      # Генерация запросов (Пуассон + Zipf)
│   ├── telemetry.py           # EWMA-сглаживание и составной индикатор нагрузки
│   ├── noise_model.py         # Гауссов шум, задержка, потери
│   ├── invariants.py          # Проверка инвариантов кольца
│   ├── metrics.py             # D(t), M_keys, агрегаты
│   └── experiment.py          # Сборка эксперимента, единый поток запросов
├── configs/                   # YAML-конфигурации серий C1–C5
├── scripts/                   # Запуск, анализ, графики
│   ├── run_all.py             # Запуск всех серий
│   ├── run_series.py          # Запуск одной серии
│   ├── run_sensitivity.py     # Серия C5: чувствительность параметров
│   ├── analyze.py             # Таблицы со средним +/- 95% ДИ, Уилкоксон
│   ├── effect_size.py         # Дельта Клиффа + boxplot/ECDF распределений
│   ├── plot_traces.py         # Графики D(t) и M_keys(t)
│   ├── build_all_results.py   # Сборка results/all_results.json
│   └── calibrate.py           # Калибровка интенсивности нагрузки Lambda
├── tests/                     # Юнит-тесты (pytest)
├── results/                   # Результаты прогонов (per-run JSON + агрегаты)
├── requirements.txt
├── LICENSE                    # MIT
└── README.md
```

## Установка

Требования: Python 3.10+, библиотеки из `requirements.txt`.

```bash
git clone https://github.com/Kronv3rk/ach-experiment.git
cd ach-experiment
python -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

## Быстрая проверка

Запуск юнит-тестов (около 30 секунд):

```bash
pytest tests/ -v
```

Должно пройти 57 тестов: проверка инвариантов кольца, гистерезиса,
бюджетного ограничения, телеметрии.

## Воспроизведение результатов статьи

### Все серии разом

```bash
python scripts/run_all.py
```

Это запустит серии C1–C5 со следующим числом независимых прогонов:

| Серия | Сценарий | Прогонов | Шагов T | Примерное время |
|-------|----------|----------|---------|-----------------|
| C1  | Стационарная нагрузка | 30 | 2000 | ~5 мин |
| C2  | Скачок + периодика + деградация узлов | 30 | 2000 | ~5 мин |
| C3  | Динамика топологии (churn) + горячие ключи | 30 | 2000 | ~6 мин |
| C4  | Зашумлённая телеметрия | 30 | 2000 | ~5 мин |
| C4a, C4b, C4c | Подсерии с разной интенсивностью шума | 30 каждая | 2000 | ~15 мин |
| C5  | Чувствительность параметров (eps_on, kappa, tau, M_max) | 10 на точку | 2000 | ~12 мин |

Полный прогон на типовой рабочей станции занимает около часа.

Все случайные семена детерминированы:
- базовое семя: 42,
- прогон номер `r` использует семя `42 + r * 1000`.

Один и тот же поток запросов подаётся всем пяти алгоритмам в одном прогоне.

### Одна серия

```bash
python scripts/run_series.py --series C1 --n-runs 30
```

Результаты пишутся в `results/C1/run_NNN.json` (один файл на прогон).

### Серия C5 (чувствительность параметров)

```bash
python scripts/run_sensitivity.py --n-runs 10
```

Результат: `results/C5/sensitivity.json`.

### Статистический анализ

После прогонов:

```bash
# Таблицы со средними +/- 95% ДИ и p-значениями Уилкоксона
python scripts/analyze.py --input-dir results

# Дельта Клиффа + распределения D̄ (boxplot + ECDF в формате SVG)
python scripts/effect_size.py --results-dir results --plots

# Графики D(t) и M_keys(t) по сериям
python scripts/plot_traces.py --input-dir results

# Сборка единого агрегата
python scripts/build_all_results.py --results-dir results
```

Файл `results/all_results.json` содержит итоговые агрегаты (средние, ДИ,
p-значения, дельты эффекта), которые цитируются в статье.

## Параметры эксперимента

Основные значения (см. таблицу 3.2 статьи):

| Параметр | Значение | Назначение |
|----------|----------|------------|
| ell\* | 0.65 | Целевой уровень загрузки |
| eps_on / eps_off | 0.10 / 0.04 | Пороги гистерезиса |
| tau / tau_min | 0.85 / 0.50 | Порог общей загрузки |
| kappa | 0.10 | Смещение чувствительности |
| M_max | 0.02 | Максимум доли ребалансировки за шаг |
| M_H / H | 0.25 / 100 | Бюджет за окно H шагов |
| v_min | 3 | Минимум токенов на узел |
| V | 1000 | Виртуальных токенов на кольце |
| NK | 50000 | Размер пространства ключей |
| n0 | 10 | Начальное число узлов |

Кластер из 10 узлов: 3 сильных (cap=1.0), 4 средних (cap=0.55), 3 слабых
(cap=0.25). Скорость обслуживания сильного узла: 350 запросов на шаг.

## Метрики

- **D(t)** — дисперсия загрузки: `max(ell) - mean(ell)`.
- **M_keys(t)** — доля пространства ключей, переназначенная за шаг t.
- **M_cum** — кумулятивная стоимость перестроек после прогревания.
- **pi_chg** — доля шагов с ненулевыми переназначениями.

Все агрегаты считаются после стартового прогрева 40 шагов.
